# ADR-0019: Adopt the bigpowers YAML spec layout

**Status:** Accepted
**Date:** 2026-06-22

## Context

The project will be built with the bigpowers skill set (scope-work, plan-release, slice-tasks, plan-work,
build-epic, session-state). Those skills read/write a structured `specs/` cockpit
(product/SCOPE*LATEST.yaml, release-plan.yaml, epics/eNN-*/ capsules, state.yaml, planning-status.yaml,
execution-status.yaml, adr/NNNN-\_.md, tech-architecture/). The original specs were free-form markdown
(SCOPE.md, RELEASE-PLAN.md) — the exact shape bigpowers itself archived as legacy.

## Decision

Migrate to the bigpowers YAML layout. SCOPE.md → product/SCOPE_LATEST.yaml; the ADR table → adr/NNNN-\*.md;
add tech-architecture/tech-stack.md; RELEASE-PLAN.md → release-plan.yaml index + archived detailed backlog
(specs/archive/RELEASE-PLAN-detailed.md) consumed by slice-tasks. Per-story epic capsules are generated by
slice-tasks/plan-work (gated), not hand-authored.

## Consequences

The bigpowers skills can drive the project end-to-end. Detailed Gherkin/Gates from the old plan are
preserved in the archived backlog and become capsule content via slice-tasks.
